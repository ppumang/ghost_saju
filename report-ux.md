# 귀신사주 UX/브랜딩 분석 보고서

---

## 1. 서비스 개요

- **서비스명**: 귀신사주 - 김귀자 할머니의 사주풀이
- **기술 스택**: Next.js 16 + React 19 + Framer Motion + Anthropic Claude API
- **컨셉**: 1990년대 경상북도 청송의 김귀자 할머니 캐릭터를 재현한 공포 분위기 사주 풀이 서비스
- **전체 플로우**: 인트로(할머니 사진 + 텍스트 애니메이션) → 생전 영상 보기 → 영상 재생(3초 후 시작 버튼) → 경고 팝업 → 생년월일/성별/음양력 입력 → 로딩(실시간 리뷰 롤링) → 결과(사주 풀이 + 리뷰 입력)

## 2. UX 플로우 강점/약점

### 2.1 강점

#### 서사적 온보딩 설계
인트로 시퀀스가 단순 입력 폼이 아니라 "이야기에 빨려들어가는 경험"으로 설계되어 있다. 할머니 사진이 먼저 페이드인되고, 텍스트가 한 줄씩 순차적으로 나타나는 방식은 영화의 오프닝 크레딧을 연상시킨다. 이는 사용자에게 "사주를 본다"는 행위에 대한 심리적 무게감을 부여하며, 일반적인 사주 앱과의 차별화 포인트가 된다.

#### 감정 곡선(Emotional Arc) 설계
검은 배경 → 영상 → 경고 팝업 → 입력 → 로딩 → **흰 배경 결과**라는 플로우에서, 마지막 결과 화면만 `#fafaf5` 흰 배경으로 전환되는 것은 의도적인 감정 대비(contrast)다. 어둠 속에서 긴장을 쌓다가 갑자기 밝은 공간에서 결과를 마주하게 함으로써, 사주 풀이 텍스트가 "계시"처럼 느껴지는 효과가 있다. 이 배경 전환은 매우 훌륭한 UX 결정이다.

#### 스킵 버튼 제공
`SkipButton` 컴포넌트가 인트로 씬에 3초 딜레이 후 나타나, 반복 방문 사용자가 인트로를 건너뛸 수 있다. 이는 "분위기"와 "효율성" 사이의 균형을 적절히 잡은 설계다.

#### 영상 프리로딩
인트로 씬에서 `<video preload="auto" style={{ display: "none" }}>`으로 영상을 미리 로드한다. 사용자가 "생전 영상 보기" 버튼을 눌렀을 때 즉시 재생이 가능하게 하는 좋은 기술적 배려다.

#### 로딩 씬의 심리적 장치
로딩 중 "실시간 리뷰"를 롤링 표시하는 것은 사회적 증거(social proof) 전략과 대기 시간 심리적 단축 효과를 동시에 달성한다. "사주를 풀고 있습니다..." → "운명의 흐름을 읽는 중..." → "귀신 사주가 말하고 있습니다..." 같은 메시지 전환은 단순 스피너보다 훨씬 효과적이다.

### 2.2 약점

#### 인트로 텍스트 과다 (13줄, 약 15초 이상 소요)
`introText.ts`의 13줄 텍스트가 순차적으로 나타나는 데 `INITIAL_DELAY(1초) + STAGGER_DELAY(0.5초) x 12줄 + EMPTY_LINE_EXTRA_DELAY(0.6초) + LINE_FADE_DURATION(0.7초) + HOLD_DURATION(2초)` = 약 10.3초에 이어 "생전 영상 보기" 버튼이 나타난다. 스킵 버튼은 3초 후 나타나지만, 처음 방문자는 전체 시퀀스를 끝까지 봐야 한다는 강박이 생긴다. 텍스트 양을 7-8줄로 줄이거나, 2-3줄씩 그룹으로 페이드인하는 방식을 고려할 필요가 있다.

#### "생전 영상 보기" 버튼과 "귀신 사주 풀이 시작하기" 버튼의 기능적 모호성
영상 씬에서 3초 후 나타나는 "귀신 사주 풀이 시작하기" 버튼이 영상 중앙(`top: 50%; left: 50%; transform: translate(-50%, -50%)`)에 오버레이된다. 이는 (a) 사용자가 영상에 집중하는 것을 방해하고, (b) "영상을 더 봐도 되는지" 불확실성을 주며, (c) 영상 콘텐츠를 가린다.

#### 에러 상태에서의 UX 부재
`useIntroSequence.ts`에서 API 호출 실패 시 `setScene("birthForm")`으로 되돌아가면서 에러 메시지를 표시하지만, 로딩 씬에서 바로 입력 폼으로 전환되면 사용자는 "무슨 일이 일어났는지" 맥락을 잃는다. 로딩 씬 내에서 에러 상태를 보여주고 "다시 시도" 버튼을 제공하는 것이 나을 수 있다.

#### 결과 화면에서 "다시 보기"의 의미 혼동
`restart` 함수가 `setScene("birthForm")`으로 돌아가는데, "다시 보기"라는 레이블은 "같은 결과를 다시 보기"로 오해될 수 있다. "새로운 사주 보기"나 "다른 사주 풀기"가 더 명확하다.

## 3. 각 단계별 이탈 포인트와 개선 방안

### 3.1 인트로 씬 (이탈 위험도: 높음)

| 이탈 원인 | 상세 | 개선 방안 |
|-----------|------|-----------|
| 텍스트 시퀀스가 너무 길다 | 13줄 순차 등장에 약 10초 소요 | 핵심 메시지 5-6줄로 압축, 나머지는 "더 알아보기"로 분리 |
| 첫 방문자의 기대 불일치 | 사주 서비스인데 즉시 입력이 불가 | 인트로 시작 전 "사주를 보시려면 잠시 기다려주세요" 같은 안내 없이 바로 어두운 화면 노출 |
| 스킵 버튼 인지 어려움 | 우측 하단에 낮은 투명도(0.5)로 배치 | 처음 5초 동안은 투명도를 높이고 잠시 애니메이션으로 주목 유도 |

### 3.2 영상 씬 (이탈 위험도: 중간)

| 이탈 원인 | 상세 | 개선 방안 |
|-----------|------|-----------|
| 자동 재생 실패 | iOS/일부 브라우저에서 오디오 있는 영상 자동재생 차단 | 최초 muted 상태로 재생 후 소리 켜기 유도 (현재 unmuted 상태로 시도) |
| 영상 길이 불확실 | 사용자가 영상이 얼마나 남았는지 알 수 없음 | 프로그레스 바 또는 "잠시 후 사주 풀이가 시작됩니다" 카운트다운 |
| 진행 버튼이 영상을 가림 | 3초 후 화면 중앙에 오버레이 | 하단에 자연스럽게 배치하거나, 영상 종료 시점에 표시 |

### 3.3 경고 팝업 (이탈 위험도: 낮음)

| 이탈 원인 | 상세 | 개선 방안 |
|-----------|------|-----------|

## 4. 브랜딩 일관성 분석

### 4.1 색상 시스템

| 요소 | 값 | 평가 |
|------|-----|------|
| 배경(인트로~로딩) | `#000000` | 공포 분위기에 적합. 일관됨 |
| 주 텍스트 | `#d4c5a9` (고서적 느낌의 금빛 베이지) | 한지/촛불 느낌을 잘 살림 |
| 글로우/강조 | `rgba(255, 69, 0, ...)` (주황-붉은 빛) | 귀화(鬼火)/도깨비불 이미지와 일치. 효과적 |
| 결과 배경 | `#fafaf5` | 의도적 대비로 훌륭하지만, 전환이 급격함 |
| 결과 강조(볼드) | `#8b0000` (다크 레드) | "XX귀신" 키워드에 사용. 공포 테마와 일관됨 |

**평가**: 색상 시스템은 매우 일관되며, 어둠(인트로~로딩) → 빛(결과)의 전환이 사주 풀이의 "계시" 느낌을 잘 전달한다. 다만 결과 화면의 급격한 밝기 전환에 0.5~1초 정도의 부드러운 트랜지션이 추가되면 더 자연스러울 것이다 (현재 `body.style.background`를 직접 변경하므로 CSS transition이 적용되지 않음).

### 4.2 타이포그래피

| 요소 | 설정 | 평가 |
|------|------|------|
| 주 폰트 | Nanum Myeongjo (나눔명조) | 전통 서적/고서 느낌으로 테마에 완벽히 부합 |
| 폴백 폰트 | Gungsuh, Batang, serif | 한글 명조체 계열 폴백으로 적절 |
| 웨이트 | 400, 700, 800 | 충분한 변화폭 |
| 사이징 | `clamp()` 사용 | 반응형 적용 양호 |

**평가**: 타이포 선택은 이 서비스의 가장 강력한 브랜딩 요소 중 하나다. 나눔명조의 전통적이고 서사적인 느낌이 "고 김귀자 할머니"라는 캐릭터와 "사주 풀이"라는 콘텐츠의 성격에 정확히 맞다. 다만 모든 UI 요소(버튼, 레이블, 드롭다운)에도 동일한 명조 폰트를 사용하고 있어, 폼 요소의 가독성이 떨어질 수 있다. 입력 UI에 한해 산세리프(고딕) 폰트를 혼용하면 기능적 구분이 명확해진다.

### 4.3 분위기/무드 일관성

| 씬 | 무드 | 일관성 |
|----|------|--------|
| 인트로 | 어둠 + 안개(fog) + 비네팅 + 텍스트 글로우 = 심령 강의장 | O |
| 영상 | 복원 영상 + 어두운 배경 | O |
| 경고 팝업 | 어둡고 절제된 모달 | O |
| 입력 폼 | 어두운 배경 + 비네팅 | O |
| 로딩 | 어두운 배경 + 펄스 텍스트 | O |
| 결과 | **갑작스러운 밝은 배경** | 의도적 전환(O), 단 전환 자체가 브랜딩 단절로 느껴질 수 있음 |

**총평**: 5개 씬에 걸쳐 색상, 폰트, 비네팅 효과, 글로우 효과가 일관되게 적용되어 있어 브랜딩 완성도가 높다. "fog drift" 애니메이션은 실제 심령 느낌을 잘 살린다. 결과 화면의 급격한 톤 변화는 의도적이지만, 트랜지션 부재로 인해 "다른 사이트에 온 것 같은" 느낌이 있을 수 있다. 결과 화면에도 미세한 비네팅이나 할머니 캐릭터 요소(작은 인장 마크 등)를 추가하면 브랜딩 연속성이 강화될 것이다.

## 5. 모바일 UX 문제점

### 5.1 영상 자동재생 이슈 (심각도: 높음)
`VideoScene.tsx`에서 영상을 자동재생(`autoPlay`)하되 `muted` 속성 없이 시도한다. iOS Safari에서는 muted가 아닌 영상의 자동재생이 차단된다. `play().catch()`로 에러를 무시하고 있지만, 사용자 관점에서는 빈 검정 화면에 3초 후 버튼만 덩그러니 나타나는 결과가 될 수 있다. **영상 자동재생 실패 시의 폴백 UI(재생 버튼 표시 등)가 반드시 필요하다.**

### 5.2 드롭다운 사용성 (심각도: 중간)
연도 선택(1920~2026, 107개 옵션), 시간 선택(48개 + 1개 옵션)이 커스텀 스타일 `<select>`로 구현되어 있다. `appearance: none`을 적용했으나, iOS/Android의 네이티브 wheel picker는 유지된다. 그러나 옵션이 과다하여 원하는 값을 찾기까지 스크롤이 많이 필요하다. 특히 시간 선택에서 "야자시초 (23:00~23:30)" 같은 긴 텍스트가 좁은 화면에서 잘릴 수 있다.

### 5.3 `position: fixed` + `overflow: hidden` 조합 (심각도: 중간)
모든 씬이 `position: fixed; inset: 0`으로 뷰포트 고정되어 있고, `body`에 `overflow: hidden`이 적용되어 있다. iOS Safari에서 가상 키보드가 올라올 때(BirthForm의 select 포커스 시) 뷰포트 크기가 변하면서 레이아웃이 밀릴 수 있다. `viewport-fit: cover`는 적용되어 있지만, 가상 키보드에 의한 뷰포트 축소는 별도 처리가 필요하다.

### 5.4 safe area 미처리 (심각도: 낮음)
`viewport-fit: cover`를 설정했지만, 실제 `env(safe-area-inset-bottom)` 등의 safe area padding은 적용되지 않았다. 노치가 있는 기기에서 하단 버튼(건너뛰기, 소리 끄기)이 홈 인디케이터에 겹칠 수 있다. 특히 `SkipButton`의 `bottom: 2rem`과 `muteButton`의 `bottom: 2rem`은 safe area를 고려해야 한다.

### 5.5 결과 화면 스크롤 (심각도: 낮음)
결과 씬에서 `body.style.overflow = "auto"`로 전환하지만, `.container`가 `position: fixed`로 유지되면서 자체적으로 `overflow-y: auto`를 갖고 있다. 이 이중 스크롤 구조가 iOS에서 "고무줄 효과(elastic bounce)"와 충돌할 수 있다.

### 5.6 터치 타겟 (심각도: 양호)
대부분의 버튼에 `min-width: 44px; min-height: 44px`가 적용되어 있어 Apple의 44pt 터치 타겟 가이드라인을 준수한다. 이 부분은 잘 구현되어 있다.

## 6. 접근성 이슈

### 6.1 색상 대비 (WCAG 미충족 가능성)
- `#d4c5a9` on `#000000` = 약 10.5:1 (AA 충족)
- 하지만 로딩 씬 리뷰의 `rgba(212, 197, 169, 0.55)` (실질적으로 `#5e5841` 수준) on `#000` = 약 3.1:1로 **AA 미충족**
- 리뷰 날짜 `rgba(212, 197, 169, 0.3)` = 약 1.8:1로 **심각한 대비 부족**
- 결과 화면 `#888` on `#fafaf5` = 약 3.5:1로 **AA 미충족 (본문 기준)**

### 6.2 스크린 리더 지원
- `aria-label`이 스킵 버튼("인트로 건너뛰기")과 음소거 버튼에 적용되어 있어 기본적인 배려가 되어 있다.
- 그러나 인트로 텍스트 시퀀스의 순차 등장이 `aria-live` 영역으로 선언되지 않아, 스크린 리더 사용자는 텍스트가 나타나는 것을 인지하지 못할 수 있다.
- 할머니 사진에 적절한 `alt` 텍스트("고 김귀자 할머니")가 있다.

### 6.3 키보드 내비게이션
- 토글 버튼(성별, 달력 기준)이 일반 `<button>`으로 구현되어 있어 키보드 접근은 가능하나, `role="radiogroup"` / `role="radio"` 같은 ARIA 역할이 없어 논리적 그룹을 인지하기 어렵다.
- 인트로의 "생전 영상 보기" 버튼이 텍스트 시퀀스 완료 전까지 DOM에 없으므로, Tab 키로 접근할 포커스 대상이 없다.

### 6.4 모션 민감도
- `prefers-reduced-motion` 미디어 쿼리가 적용되어 있지 않다. 안개 드리프트, 텍스트 펄스, 글로우 애니메이션 등이 전정 기능 장애(vestibular disorder)가 있는 사용자에게 불편을 줄 수 있다.
- Framer Motion의 `transition` 값들이 하드코딩되어 있어, 시스템의 "모션 줄이기" 설정을 존중하지 않는다.

### 6.5 포커스 관리
- 씬 전환 시 포커스가 자동으로 새 씬의 첫 인터랙티브 요소로 이동하지 않는다. AnimatePresence의 exit/enter 사이에 포커스가 사라질 수 있다.

## 7. 경쟁 서비스 대비 차별화 포인트

### 7.1 주요 경쟁 서비스와의 비교

| 비교 항목 | 만세력 / 일반 사주 앱 | 신한점 / 운세 서비스 | 귀신사주 |
|-----------|----------------------|---------------------|---------|
| **UX 컨셉** | 실용적/정보 전달 중심 | 깔끔한 모던 UI | 공포 엔터테인먼트 + 사주 |
| **캐릭터** | 없음 | 없거나 귀여운 마스코트 | 실존 인물 기반 (고 김귀자 할머니) |
| **콘텐츠 형식** | 항목별 나열 (성격, 재물, 건강 등) | 카드 형식 / 짧은 문장 | 소설적 서사 (2000자 분량 자서전 스타일) |
| **감정적 경험** | 중립적 / 정보적 | 가볍고 긍정적 | 공포 → 긴장 → 카타르시스 |
| **소셜 바이럴 요소** | 낮음 | 공유 기능 탑재 | 현재 없음 (최대 약점) |
| **재방문 동기** | 일운/월운 확인 | 매일 운세 | 현재 낮음 |

### 7.2 핵심 차별화 포인트

1. **실존 인물 기반 스토리텔링**: "고 김귀자 할머니"라는 실재했던 인물의 이야기와 캐릭터를 기반으로 한 서비스는 사주/운세 시장에서 유일무이하다. 이는 "리얼리티"와 "공포"를 동시에 충족하며, 단순한 알고리즘 기반 서비스와 확연히 구별된다.

2. **공포 장르와 사주의 결합**: 사주 풀이를 엔터테인먼트 콘텐츠로 포장한 접근은 기존 시장에 없는 포지셔닝이다. "무서워서 못 하겠다"가 오히려 바이럴 포인트가 될 수 있다.

3. **서사적 결과 포맷**: 항목별 나열이 아닌 "한 편의 이야기"로 사주를 전달하는 방식은 읽는 재미가 있고, 스크린샷 공유 시 흥미를 유발한다. 특히 "1부(경고/공포) → 2부(희망/잘 풀렸을 때)"의 양면 구조는 감정적 완결성을 제공한다.

4. **"XX귀신" 키워드 시스템**: 결과에 포함되는 **돈귀신**, **여자귀신**, **명예귀신** 같은 키워드는 사용자의 약점을 기억에 남는 형태로 전달하면서, "나는 XX귀신이래" 라는 자연스러운 대화/공유 소재를 만든다. 이는 매우 바이럴한 요소다.

## 8. 종합 개선 우선순위

### 즉시 개선 (HIGH)
1. **VideoScene 자동재생 muted 처리** + 재생 실패 폴백 UI
2. **SNS 공유 기능** 추가 (카카오톡, 링크 복사, 결과 이미지화)
3. **로딩 리뷰 하드코딩 → 동적 데이터** 또는 날짜 자동 생성
4. **safe area padding** 적용 (하단 버튼들)

### 단기 개선 (MEDIUM)
5. 인트로 텍스트 분량 축소 또는 단락 그룹화
6. 시간 입력 UX 간소화 (기본 모드: 12지지 2시간 단위 / 상세 모드: 30분 단위)
7. 연도 입력을 숫자 직접 입력으로 변경
8. `prefers-reduced-motion` 미디어 쿼리 적용
9. 결과 → 인트로 배경 전환에 CSS transition 추가

### 장기 개선 (LOW)
10. 결과 PDF 또는 이미지 저장 기능
11. ARIA 역할(radiogroup 등) 및 포커스 관리 강화
12. 실제 사용자 리뷰 DB 연동 (로딩 씬에 실시간 반영)
13. 재방문 동기 설계 (일운/월운 추가 콘텐츠)

## 9. 결론

귀신사주는 "공포 엔터테인먼트 + 사주 풀이"라는 독창적 포지셔닝과 실존 인물 기반 스토리텔링, 높은 수준의 비주얼 브랜딩 일관성을 갖춘 서비스다. 특히 인트로에서 결과까지의 감정 곡선 설계와 나눔명조를 중심으로 한 타이포그래피 시스템은 경쟁 서비스 대비 확실한 차별점이다.

가장 시급한 개선 포인트는 (1) 모바일 영상 재생 안정성, (2) SNS 공유 기능 추가를 통한 바이럴 루프 구축, (3) 입력 폼의 모바일 사용성 개선이다. "XX귀신" 키워드와 공포 분위기라는 강력한 바이럴 소재가 있음에도 공유 기능이 없는 것이 현재 가장 큰 성장 병목이다.
