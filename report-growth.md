## 1. 현재 수익 모델 분석

### 현재 상태: 수익 모델 부재

현재 코드베이스를 분석한 결과, **실질적인 수익 모델이 전혀 구현되어 있지 않다.**

#### 현재 구현된 것
- 무료 사주 풀이 제공 (Claude Sonnet 4.5 API 호출)
- 리뷰 수집 시스템 (`/api/review` 엔드포인트, `reviews.json` 파일 저장)
- "사주 파일은 pdf로 제공됩니다"라는 문구 (`LoadingScene.tsx` 88행)만 존재하고 **실제 PDF 생성/다운로드 기능은 미구현**

#### 핵심 문제점
1. **모든 기능이 완전 무료**: 사주 풀이를 무제한으로 받을 수 있고, 어떠한 결제 장벽도 없음
2. **PDF 기능 허수**: 로딩 화면에 "pdf로 제공됩니다"라고 명시했으나 결과 화면(`ResultScene.tsx`)에는 PDF 다운로드 버튼이나 관련 로직이 전혀 없음. 이는 사용자 기대를 형성하지만 충족시키지 못하는 상태
3. **유저 데이터 미활용**: 생년월일, 성별 등 수집된 데이터를 분석이나 마케팅에 활용하는 구조 없음 (개인정보 관련 고지/동의도 없음)
4. **트래픽 모니터링 부재**: Google Analytics 등 분석 도구 미연동

## 2. 구체적 수익화 방안

### 2-1. 프리미엄 사주 (핵심 수익 모델, 권장)

| 구분 | 무료 (현재) | 프리미엄 (신규) |
|------|-----------|---------------|
| 사주 풀이 | 1부만 (어두운 서사, ~600자) | 1부 + 2부 전체 (2,000자) |
| XX귀신 키워드 | 가려서 표시 ("??귀신") | 전체 공개 |
| PDF 저장 | 불가 | 고급 디자인 PDF 다운로드 |
| 가격 | 무료 | 1,900~3,900원 |

**구현 전략**: 현재 API 응답을 1부/2부로 분리하고, 무료 사용자에게는 1부의 클라이맥스(귀신 키워드) 직전에서 끊어 호기심을 극대화. "니 사주에 붙은 귀신이 뭔지 알고 싶으면..."이라는 전환 문구로 결제 유도.

**예상 전환율**: 무료 -> 유료 5~10% (공포+호기심 조합의 강력한 심리적 드라이버)

### 2-2. 궁합 사주 (확장 상품)

- 두 사람의 사주를 입력받아 궁합을 귀신사주 스타일로 풀이
- 가격: 3,900~5,900원
- 연인/부부/친구 궁합 등 카테고리 세분화
- "니랑 그 사람 사이에 낀 귀신이 있다..." 같은 공포 궁합 콘셉트

### 2-3. 2025년 신년운세 (시즌 상품)

- 매년 연말~설 시즌 한정 상품
- 해당 연도 월별 운세를 귀신사주 스타일로 제공
- 가격: 4,900~9,900원
- 시즌성이 강해 FOMO(놓칠까 봐 두려운 심리) 활용 가능

### 2-4. 카카오페이/토스 결제 연동

- 한국 시장 특성상 간편결제 필수
- 토스페이먼츠 or 포트원(구 아임포트) 연동
- 소액결제에 최적화된 UX (1~2번 탭으로 결제 완료)

### 2-5. 광고 수익 (보조 수익)

- 결과 페이지 하단에 네이티브 광고 삽입
- Google AdSense or 카카오 애드핏
- 주의: 공포 분위기를 깨지 않는 광고 배치 필요
- 예상 RPM: 1,000~3,000원 (운세/사주 카테고리는 광고 단가가 높은 편)

### 2-6. 부적/액막이 굿즈 (e-커머스 확장)

- 사주 결과에 맞는 디지털 부적 이미지 판매 (1,000~2,000원)
- 스마트폰 배경화면용 액막이 부적
- 물리적 부적은 파트너십으로 (제작 리스크 회피)

## 3. 타겟 오디언스 및 페르소나

### 핵심 타겟 (Primary)

**페르소나 1: "호기심 MZ" (20대 중반~30대 초반)**
- 사주/운세에 깊은 신념은 없지만 "재미로" 보는 층
- SNS 공유에 적극적 (인스타 스토리, 틱톡 리액션)
- 친구들과 함께 보며 소름 공유하는 것을 즐김
- 소비 패턴: 소액(2,000~5,000원) 즉흥 결제에 저항감 낮음
- 접점: 틱톡, 인스타그램, 트위터(X), 에브리타임

**페르소나 2: "운세 매니아" (25~40대 여성)**
- 연초/생일마다 사주, 타로, 운세를 정기적으로 확인
- 신한라이프 사주, 카카오 운세 등 이미 경험한 사용자
- 차별화된 사주 서비스에 대한 수요 존재
- 소비 패턴: 운세 관련 유료 결제 경험 있음, 가격 민감도 낮음
- 접점: 네이버 카페, 운세 앱 사용자, 맘카페

**페르소나 3: "공포 콘텐츠 팬" (18~30대)**
- 호러 웹툰, 괴담, 공포 유튜브 등을 즐기는 층
- "사주+공포"라는 신선한 조합에 끌림
- 새벽에 혼자 보며 소름 끼치는 경험을 원함
- 소비 패턴: 콘텐츠 과금에 익숙 (웹툰, 스트리밍 등)
- 접점: 유튜브 공포 채널, 레딧 nosleep, 에펨코리아 괴담 게시판

### 확장 타겟 (Secondary)

**페르소나 4: "경상도 향수" (30~50대)**
- 경상도 사투리와 토속 문화에 향수를 느끼는 층
- "우리 할머니도 이런 식으로 말씀하셨다"는 정서적 연결
- 가족/부모님 사주를 봐드리는 효도 소비

## 4. SNS 바이럴 마케팅 전략

### 4-1. 틱톡 / 인스타 릴스 전략

**포맷 A: "내 사주에 붙은 귀신" 리액션 영상**
- 사용자가 사주 결과의 XX귀신 키워드를 보고 놀라는 리액션
- "#귀신사주 #내사주귀신 #귀신사주챌린지" 해시태그
- 숏폼 영상 15~30초: "나 귀신사주 해봤는데... [스크롤] ...돈귀신이래" + 놀라는 표정
- **예상 바이럴 지수: 최상** - 결과가 개인마다 다르므로 반복 콘텐츠 생산 가능

**포맷 B: "새벽 3시에 귀신사주 해보기" 챌린지**
- 공포 분위기를 극대화한 ASMR식 촬영
- 어두운 방에서 폰 화면만 비추며 사주 결과를 읽는 포맷
- 공포 채널 크리에이터와 콜라보

**포맷 C: "부모님 사주 봐드리기" 감성 콘텐츠**
- 부모님 생년월일로 사주 풀이 후 부모님께 보여드리는 리액션
- 경상도 사투리에 부모님이 반응하는 장면은 감성+웃음 요소
- 부모님 세대에게로 서비스 확산 가능

### 4-2. 트위터(X) 전략

**스레드형 콘텐츠**
- "귀신사주로 본 1995년생의 운명..." 같은 출생연도별 시리즈
- 사주 결과 일부를 캡처해서 공유하는 문화 형성
- 결과 페이지에 트위터 공유 버튼 추가 (현재 미구현)

**밈 생성**
- "XX귀신" 키워드를 활용한 자기 비하 유머
- "돈귀신이래... 통장잔고 보니까 맞는 듯"
- "여자귀신... 솔로 6년차인데 어디서 나타날 여자귀신?"

### 4-3. 유튜브 전략

**공포 유튜버 협업**
- 곽튜브, 떵개떵 등 공포/괴담 채널과 콜라보
- "실제 존재했던 귀신사주 할머니의 사주풀이를 AI로 재현했다" 스토리텔링
- 김귀자 할머니의 생전 영상 소재 활용 (현재 `restored-video.mp4` 보유)

**ASMR / 분위기 영상**
- 귀신사주 풀이 과정을 ASMR 스타일로 촬영
- 경상도 사투리 낭독 + 새벽 분위기 조명

### 4-4. 커뮤니티 시딩 전략

| 플랫폼 | 시딩 포맷 | 예상 효과 |
|--------|---------|---------|
| 에브리타임 | "아까 귀신사주 해봤는데 소름..." 후기글 | 대학생 사이 급속 확산 |
| 디시인사이드 | 귀신사주 인증글 + XX귀신 결과 공유 | 밈화 가능성 |
| 네이버 카페 | 운세/사주 카페에 후기 작성 | 운세 매니아 유입 |
| 카카오톡 | 결과 공유 링크 (OG 이미지 최적화 필요) | 1:1 구전 효과 |
| 레딧 | r/korea, r/nosleep 한국판 | 해외 한인 유입 |

### 4-5. 공유 기능 구현 (현재 미구현, 최우선 필요)

현재 결과 페이지(`ResultScene.tsx`)에 **공유 버튼이 전혀 없다.** 바이럴의 가장 기본적인 장치가 빠져 있으므로 다음을 즉시 추가해야 한다:

1. **카카오톡 공유**: 한국 시장 1순위. OG 이미지에 "귀신사주" 로고 + XX귀신 키워드 포함
2. **인스타그램 스토리 공유**: 사주 결과를 카드 이미지로 변환하여 스토리 공유
3. **트위터 공유**: "나의 귀신사주 결과: XX귀신 - [URL]" 형태의 원클릭 공유
4. **링크 복사**: 범용 공유
5. **결과 카드 이미지 생성**: html2canvas 등으로 결과를 이미지화하여 저장/공유 가능하게

## 5. 경쟁 분석

### 5-1. 주요 경쟁사 비교

| 항목 | 신한라이프 사주 | 카카오 운세 | 점신 | **귀신사주** |
|------|-------------|----------|------|-----------|
| 가격 | 무료 (보험 마케팅) | 무료/유료 혼합 | 무료/유료 | 현재 무료 |
| 차별점 | 보험사 신뢰 + 심플 UI | 플랫폼 파워 | 전문 사주앱 | **공포+사투리+캐릭터** |
| 톤 | 밝고 긍정적 | 중립적, 범용적 | 전통적, 진지 | **어둡고 으스스** |
| 캐릭터 | 없음 | 없음 | 없음 | **김귀자 할머니** |
| 바이럴성 | 높음 (보험사 규모) | 보통 | 낮음 | **매우 높음 (잠재)** |
| 리텐션 | 낮음 (일회성) | 중간 (습관성) | 높음 (다양한 메뉴) | 낮음 (현재) |

### 5-2. 귀신사주의 핵심 차별점 (MOAT)

1. **유일무이한 포지셔닝**: "공포 사주"라는 카테고리 자체를 창출. 기존 사주 서비스는 모두 "밝고 긍정적" 또는 "전통적이고 진지한" 톤인 반면, 귀신사주는 **공포+엔터테인먼트**라는 블루오션
2. **캐릭터 IP**: 김귀자 할머니라는 구체적 인물 기반의 캐릭터. 실존 인물의 생전 영상까지 보유하여 스토리텔링의 신뢰도가 높음
3. **경상도 사투리**: 표준어가 아닌 사투리 사용은 차별화이자 정서적 친밀감 형성
4. **XX귀신 시스템**: "돈귀신", "여자귀신" 등 한 단어로 압축된 결과 키워드는 SNS 공유에 최적화. 짧고 강렬하며 밈이 되기 좋은 구조
5. **서사적 풀이 구조**: 항목별 나열이 아닌 자서전식 스토리텔링은 읽는 재미를 제공

### 5-3. 경쟁사 대비 약점 및 보완점

| 약점 | 설명 | 보완 방안 |
|------|------|---------|
| 브랜드 인지도 | 신규 서비스로 인지도 제로 | 바이럴 마케팅 집중 투자 |
| 콘텐츠 다양성 | 사주 풀이 1종만 존재 | 궁합, 연운, 토정비결 등 확장 |
| 플랫폼 파워 | 독립 웹앱으로 유입 채널 제한 | 카카오 미니앱, 네이버 스마트스토어 연동 |
| 명리학 정확도 | AI 기반으로 정통 명리학 관점 차이 존재 | 면책 조항 강화 (현재 disclaimer 팝업은 존재) |
| 반복 방문 동기 | 동일 생년월일은 항상 같은 결과 | 일운/월운 등 시간 기반 콘텐츠 추가 |

## 6. 그로스 퍼널 분석 및 최적화

### 6-1. 현재 퍼널 구조

```
[인트로 씬] → [생전 영상] → [생년월일 입력] → [로딩+리뷰] → [결과]
(introText)   (video)       (BirthForm)        (LoadingScene)   (ResultScene)
```

### 6-2. 퍼널 단계별 이탈 분석 및 최적화

**단계 1: 인트로 씬 (예상 이탈률: 40~60%)**

- 문제: 텍스트 14줄이 순차적으로 나타나며, 전체 약 15초 이상 소요. 모바일 사용자의 인내심 한계 초과
- 건너뛰기 버튼이 3초 후에 나타나지만, 우측 하단에 작게 위치하여 인지율 낮음
- 최적화안:
  - 인트로 시퀀스를 5~7초로 단축 (핵심 3줄만 유지)
  - 또는 건너뛰기 버튼을 즉시 표시하고 더 눈에 띄게 배치
  - 첫 방문 후에는 인트로 자동 스킵 (localStorage 활용)

**단계 2: 생전 영상 (예상 이탈률: 20~30%)**

- 문제: 영상의 목적과 길이가 불명확. 루프 재생으로 끝이 없음
- "귀신 사주 풀이 시작하기" 버튼이 영상 위에 겹쳐 나타나는데, 3초 딜레이가 있어 사용자가 버튼 없이 당황할 수 있음
- 최적화안:
  - 영상을 15~30초로 편집하여 자동 전환
  - 또는 영상 아래에 CTA를 항상 표시

**단계 3: 생년월일 입력 (예상 이탈률: 15~25%)**

- 문제: 입력 필드가 7~8개(성별, 달력기준, 윤달, 연도, 월, 일, 시간)로 많음
- 특히 "태어난 시간" 선택지가 55개(야자시초~해시4각)로 전통 시진 체계는 일반 사용자에게 낯설고 부담
- 최적화안:
  - 시간 선택을 "대략적 시간대" (예: 새벽 1~3시)로 간소화하고, 정밀 시진은 선택사항으로
  - 프로그레스 바 추가로 완료까지의 거리감 제공
  - "모름" 옵션을 더 눈에 띄게 배치

**단계 4: 로딩 화면 (예상 이탈률: 10~15%)**

- 문제: Claude API 응답 대기 시간이 10~30초 이상 소요될 수 있음
- 잘 설계된 부분: 로딩 중 가짜 리뷰를 보여주는 것은 이탈 방지에 효과적
- 개선 필요: "사주 파일은 pdf로 제공됩니다" 문구가 있으나 실제 PDF 미제공으로 신뢰 하락
- 최적화안:
  - 로딩 시간 예상 표시 (예: "약 15초 남음")
  - PDF 문구 제거 또는 실제 PDF 기능 구현
  - 로딩 중 공포 분위기 강화 (화면 깜빡임, 소리 효과 등)

**단계 5: 결과 화면 (전환 최적화 핵심)**

- 현재: 결과 전문 표시 + 리뷰 작성 + 다시 보기 버튼
- 문제점:
  - 공유 버튼 없음 (바이럴 차단)
  - PDF 다운로드 없음
  - 추가 상품 제안 없음 (업셀 기회 낭비)
  - "다시 보기" 버튼만 있어 이탈 후 재방문 동기 부족
- 최적화안:
  - SNS 공유 버튼 4종 추가 (카카오, 인스타, 트위터, 링크 복사)
  - "궁합 사주 보기", "2025 운세 보기" 등 추가 상품 CTA
  - 결과를 이미지 카드로 저장하는 기능
  - "친구에게 귀신사주 선물하기" 버튼

### 6-3. 퍼널 전체 최적화 요약

| 지표 | 현재 추정 | 최적화 후 목표 |
|------|---------|-------------|
| 인트로 완주율 | 40~60% | 75~85% |
| 영상 → 폼 전환 | 70~80% | 90%+ |
| 폼 완성률 | 75~85% | 90%+ |
| 로딩 이탈률 | 10~15% | 5% 이하 |
| 결과 → 공유율 | 0% (기능 없음) | 15~25% |
| 전체 전환율 (방문→결과) | ~20% 추정 | 50~60% 목표 |

## 7. 리텐션 전략

### 7-1. 현재 리텐션 구조의 문제

사주 풀이는 본질적으로 **일회성 콘텐츠**이다. 같은 생년월일을 입력하면 유사한 결과가 나오므로, 한 번 본 사용자가 다시 방문할 이유가 약하다.

### 7-2. 리텐션 강화 전략

**전략 1: 시간 기반 콘텐츠**
- 월운: 매월 1일 해당 사주의 이달 운세 제공
- 일진: 오늘의 사주 한 줄 (김귀자 할머니 스타일)
- 절기별 운세: 24절기에 맞춘 특별 풀이

**전략 2: 콘텐츠 확장으로 재방문 유도**
- 궁합 사주: 파트너 바뀔 때마다 재이용
- 이름풀이: 같은 캐릭터로 작명/이름 해석
- 꿈해몽: "김귀자 할머니의 꿈 해석"
- 관상/수상: 사진 업로드 기반 (미래 확장)

**전략 3: 커뮤니티 구축**
- XX귀신별 게시판/그룹 (돈귀신 모임, 여자귀신 모임 등)
- 사주 결과 공유 및 토론 공간
- "같은 사주끼리" 매칭

**전략 4: 알림 시스템**
- 카카오 알림톡: "올해 조심해야 할 달이 다가옵니다..."
- 푸시 알림: "김귀자 할머니가 전하는 이번 주 경고"
- 이메일 뉴스레터: 주간 귀신사주 운세

**전략 5: 이벤트 드리븐**
- 특정 날짜(한식, 백중, 추석 등) 시즌 이벤트
- "귀신의 날" (매월 음력 보름) 특별 사주
- 새해/설날 신년운세 대규모 프로모션

### 10-1. 법적/윤리적 리스크

| 리스크 | 심각도 | 대응 방안 |
|--------|------|---------|

| 미신/점술 관련 규제 | 중간 | "엔터테인먼트 목적"임을 명확히 고지. 현재 disclaimer 팝업 존재하나 법적 면책 문구 강화 필요 |
| 개인정보보호법 위반 | **높음** | 생년월일+성별 수집 시 개인정보처리방침 고지 및 동의 절차 필수 (현재 미구현) |
| 공포 콘텐츠 심약자 문제 | 중간 | 서비스 진입 시 "공포 요소 포함" 경고 추가, 연령 확인 |
| 사주 정확도 분쟁 | 낮음 | 현재 disclaimer에서 "토속적 관점" 고지 중. 충분하나 문구 보강 권장 |

### 10-2. 기술적 리스크

| 리스크 | 심각도 | 대응 방안 |
|--------|------|---------|
| Claude API 비용 폭증 | **높음** | Rate limiting 구현, 무료 사용 횟수 제한, 캐싱 도입 |
| Claude API 장애/중단 | 중간 | 에러 핸들링 개선 (현재 기본 에러 메시지만 존재), 대체 모델 준비 |
