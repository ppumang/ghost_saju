## 1. 인트로 텍스트 카피 품질 분석 (`src/data/introText.ts`)

### 현재 카피

```
해당 사주는
1990년대 경상북도 청송에서
귀신 사주라는 이름으로 이름을 날렸던
고 김귀자 할머니 (1920~2007)의 생전 사주 풀이를 바탕으로
제작되었습니다.

김귀자 할머니는 단순 사주 풀이를 넘어
해당 사주가 잘 풀리면 어떤 모습인지,
안 풀리면 어떤 모습인지를 풀어내
특정 사주에 악운을 피하고 천운을 만나는 조언을 해줄 수 있는 사주,
귀신을 피하는 사주,
귀신 같이 정확한 사주 등으로 해석되는
귀신사주로 유명하셨습니다.
```

### 평가: B+

**강점:**
- "고 김귀자 할머니 (1920~2007)"라는 실존 인물 설정이 강력한 몰입 장치로 작동한다. 생몰년까지 구체적으로 명시하여 다큐멘터리적 사실감을 부여한다.
- "1990년대 경상북도 청송"이라는 시공간 설정이 한국 토속 공포의 정서와 정확히 맞아떨어진다. 시골 + 90년대라는 조합은 "괴담"의 황금기와 겹치며 노스탤지어를 자극한다.
- "귀신사주"라는 네이밍의 중의적 해석(귀신을 피하는 / 귀신같이 정확한 / 귀신의 사주)을 잘 풀어낸다.

**약점:**
- 전체 톤이 "설명문"에 가깝다. 인트로는 서비스의 첫인상인데, 현재 카피는 할머니에 대한 정보 전달에 집중하고 있어 감정적 훅이 약하다.
- "해당 사주는"으로 시작하는 것이 공문서 같은 톤을 만든다. 사용자를 끌어들이는 힘이 부족하다.
- 두 번째 문단이 너무 길고 "~사주, ~사주, ~사주"의 반복이 읽기 지루하다.
- "이름으로 이름을 날렸던"이 어색한 동어반복이다.

### 개선 제안

인트로를 감정 중심으로 재구성할 것을 권한다. 정보 전달보다 분위기 조성이 우선이다.

```
1990년대, 경상북도 청송.

사람들은 그 할머니를 찾아
산길을 넘고 또 넘었다.

고 김귀자 (1920~2007)

니 사주에 붙은 귀신을 떼주는 사주,
니 인생이 어찌 풀릴지 귀신같이 맞추는 사주.

사람들은 그걸 이리 불렀다.

"귀신사주"
```

이 구조는 (1) 장소 설정 -> (2) 사람들의 행동으로 신비감 조성 -> (3) 인물 소개 -> (4) 핵심 개념 -> (5) 한 단어 임팩트로 마무리하는 서사 구조다. 현재의 "설명문"을 "이야기"로 바꾸면 공포 서비스 진입의 기대감이 크게 상승한다.

## 2. AI 프롬프트 품질 분석 (`src/app/api/fortune/route.ts`)

### 평가: A-

이 프로젝트의 핵심 엔진인 시스템 프롬프트는 전체적으로 높은 수준이다.

### 캐릭터 재현도: A

**강점:**
- 경상도 사투리 예시("~했다 아이가", "~하모", "그기 참...", "마 이래 보이도")가 구체적이다. AI에게 단순히 "사투리를 써라"가 아니라 예시를 주는 것은 프롬프트 엔지니어링의 핵심이다.
- "반말" 강제 + "네 사주는..."으로 시작하도록 고정한 것이 좋다. 첫 문장의 톤이 전체를 결정하기 때문이다.
- "따뜻하면서도 으스스한" 이라는 이중 톤 지시가 캐릭터의 핵심을 관통한다. 할머니라는 존재는 원래 위안과 공포의 양면성을 가진다.

**개선 포인트:**
- 사투리 예시가 4개로 한정적이다. AI가 같은 패턴만 반복 사용할 위험이 있다. "~안카나", "~데이", "~했는기라", "우야꼬", "와 이래 됐노" 등을 추가하면 다양성이 올라간다.
- 할머니의 연령대(70대 노인)에 맞는 어휘 지시가 없다. "요즘 말 대신 옛 표현을 써라 (예: '핸드폰'이 아니라 '전화기')" 같은 보조 지시가 있으면 캐릭터 일관성이 강화된다.

### 공포 요소: A-

**강점:**
- "새벽 3시에 등 뒤에서 느껴지는 싸한 기운, 꿈에서 자꾸 나타나는 그림자, 혼자 있을 때 들리는 소리, 거울에 비치는 것 같은 느낌" - 구체적 공포 시나리오 예시가 훌륭하다. 추상적인 "무섭게 써라"가 아니라 감각별 공포를 지정한다.
- "읽는 사람이 소름이 끼칠 정도로 구체적으로 묘사해라"는 강도 지시가 명확하다.
- "XX귀신" 키워드 시스템이 독창적이다. 개인화된 공포 라벨링은 바이럴 요소로도 강력하다.

**개선 포인트:**
- 공포의 그라데이션이 없다. 현재는 "처음부터 무섭게"인데, 서서히 으스스해지다가 클라이맥스에서 소름끼치는 구조가 더 효과적이다. "처음에는 약하게 암시하고, 중반에 직접적 묘사를 넣고, 조심해야 할 것 경고에서 가장 강한 공포를 배치해라"와 같은 점층적 지시를 추가할 수 있다.
- 공포의 개인화가 부족하다. "해당 사주의 약한 오행과 연결하여 어떤 종류의 귀신에게 끌리기 쉬운지를 설명해라" 같은 지시가 있으면 "이게 내 이야기"라는 느낌이 배가된다.

### 서사 구조: A

**강점:**
- "어린 시절 -> 성장기 -> 중년 -> 말년"의 자서전적 서사 구조가 명확하다. 이것은 사주 풀이에서 가장 중요한 요소다. 사람들은 "올해 운세"보다 "내 인생 전체의 이야기"에 훨씬 강하게 반응한다.
- "자서전을 읽는 느낌", "한 사람의 인생 이야기를 들려주듯이"라는 문체 지시가 항목별 나열을 방지한다.

**개선 포인트:**
- "어떤 고비나 기회를 만나는지"에서 구체적 인생 영역(연애/결혼, 직업, 건강, 재물)을 명시하면 빠뜨리는 영역이 줄어든다.

### 1부/2부 전환: A-

**강점:**
- "근데 말이다... 니 사주가 잘 풀리면 말이지..." 같은 전환 문장 예시가 있어 1부 공포 -> 2부 희망의 극적 대비가 명확하다.
- 같은 사주를 두 갈래로 보여주는 구조 자체가 독창적이며, 단순 운세와 차별화된다.
- 1부 1200자 / 2부 800자의 비율이 적절하다. 공포가 더 길어야 임팩트가 있고, 희망은 짧지만 강렬해야 여운이 남는다.

**개선 포인트:**
- 2부 마무리 예시("그러니까 조심해라... 니 사주는 잘 타고났다.")가 한 가지뿐이다. AI가 이 문장을 거의 그대로 반복할 위험이 있다. 2~3개의 변형 예시를 추가하면 좋다.

### 명리학적 정확성 지시: B+

**강점:**
- 천간/지지/오행 활용, 양력-음력 변환, 윤달 처리, 대운 순행/역행까지 명리학의 핵심 요소를 지시한다.
- "모름" 시간 옵션이 있어 시주를 모르는 사용자도 이용 가능하다.

**개선 포인트:**
- Claude가 정확한 만세력 변환을 수행하는 것은 한계가 있다. 음양력 변환과 천간지지 산출의 정확도를 높이려면 외부 만세력 API를 통해 사주 정보를 사전에 산출한 뒤 프롬프트에 삽입하는 것이 이상적이다. 현재 구조에서는 AI의 추론에 의존하므로 간혹 사주팔자 자체가 부정확할 수 있다.

## 3. 로딩 화면 리뷰 콘텐츠 (`src/components/LoadingScene.tsx`)

### 평가: A-

### 사실성/설득력 분석

**강점 - 리뷰의 자연스러움:**
- "ㄷㄷ", "ㅋㅋㅋ", "ㅠㅠ", ";;"같은 이모티콘/축약어 사용이 실제 한국 온라인 리뷰 말투와 정확히 일치한다.
- 리뷰 톤의 분포가 자연스럽다: 극찬(8개), 긍정적이되 약간의 지적(4개), 중립적 비판(3개), 부정적 의견(2개). 100% 극찬이 아니라 비판 리뷰("글이 너무 길다", "과하게 공포 조장하는 느낌")가 포함된 것이 핵심이다. 이것이 "내돈내산" 느낌의 진정성을 만든다.
- 구체적 상황 언급이 신뢰도를 높인다: "엄마 사주도 봐드렸는데", "친구들이랑 같이 봤는데", "새벽에 혼자 봤는데" 등 이용 상황이 다양하다.

**강점 - 전략적 리뷰 설계:**
- 각 리뷰가 서비스의 핵심 경험 포인트를 하나씩 강조한다:
  - "소름돋게 정확하다" -> 정확성
  - "할머니 말투가 진짜 생생해서" -> 캐릭터 몰입감
  - "재물운 부분 소름" -> 구체적 운세 영역
  - "잘 풀렸을 때 인생 파트" -> 2부 구조 가치
  - "엄마 사주도 봐드렸는데" -> 주변인 공유 유도
  - "자꾸 보게 됨" -> 재방문 암시
- 마케팅 관점에서 리뷰가 사용자의 기대치를 사전에 세팅하는 역할을 훌륭히 수행한다.

**약점:**
- 날짜가 "2025.02.05~2025.02.12"로 하드코딩되어 있다. 시간이 지나면 오래된 리뷰가 되어 신뢰도가 급락한다. 현재 날짜 기준으로 동적 생성하거나, 최소한 "최근"이라는 상대적 시간 표현을 사용해야 한다.
- "사주 파일은 pdf로 제공됩니다"라는 문구가 있는데, 실제 서비스에서 PDF 제공 기능은 없다. 사실과 다른 정보는 즉시 제거해야 한다.
- 17개 리뷰 중 "김**, 이**, 박**, 최**, 정**, 강**, 윤**, 한**, 서**, 장**, 조**, 임**, 오**, 배**, 노**, 문**, 송**" - 성씨가 모두 다르다. 실제 한국에서는 김/이/박이 절대적으로 많으므로 성씨 중복이 자연스럽다.
- 리뷰가 2초마다 교체되는 것은 다소 빠르다. 읽는 시간을 고려하면 3~4초가 적절하다.

### 로딩 메시지 분석

"사주를 풀고 있습니다..." / "운명의 흐름을 읽는 중..." / "귀신 사주가 말하고 있습니다..." - 3개가 순환하며 분위기를 유지한다. 메시지 수가 적어 반복감이 들 수 있다. "천간지지를 살피는 중...", "오행의 흐름을 따라가는 중...", "할머니가 사주를 들여다보고 있습니다..." 등을 추가하면 대기 시간의 지루함이 줄어든다.

## 4. 경고 팝업 카피 (BirthForm.tsx 내 disclaimer)

### 현재 카피

```
해당 사주풀이는 정통 명리학 기반 사주와 약간의 개인적 해석에 차이가 존재하며,
특히 일운을 보는 부분에 있어서는 토속적인 관점이 가미되었음을 미리 알려드립니다.

정통 사주학 관점의 풀이를 보시는 분들은 다른 곳으로의 방문을 추천드립니다.
```

### 평가: B

**강점:**
- "정통 명리학과 다르다"는 선제적 고지가 사주 전문가/마니아의 비판을 사전에 차단한다.
- "토속적인 관점"이라는 표현이 오히려 차별화 포인트로 작동한다. "우리는 정통이 아니라 토속이다"라는 포지셔닝이 독특하다.
- "다른 곳으로의 방문을 추천드립니다"가 역설적으로 자신감을 보여주면서 서비스 정체성을 강화한다.

**약점:**
- 경고 팝업이 공포 서비스의 톤에 맞지 않는 "공손한 안내문" 어투다. 서비스 전체가 공포-할머니 캐릭터인데 팝업만 CS 담당자가 쓴 것 같다.
- 법적 면책(entertainment disclaimer)이 없다. "본 서비스는 오락 목적이며 전문적인 상담을 대체하지 않습니다"류의 문구가 필요하다.
- "일운을 보는 부분에 있어서는"이라는 표현이 일반 사용자에게는 이해하기 어렵다.

### 개선 제안

세계관에 맞는 톤 + 필수 면책 사항 결합:

```
할머니의 사주풀이는 정통 명리학과는 다른 길을 걷습니다.
경상북도 청송의 토속 사주 관점이 담겨 있어
해석의 차이가 있을 수 있습니다.

본 서비스는 재미와 흥미를 위한 것이며,
전문적인 상담이나 의학적 조언을 대체하지 않습니다.
```

이렇게 하면 세계관 유지 + 법적 면책이 동시에 해결된다.

## 5. 바이럴 가능성 분석

### 캡처/공유 유발 요소 평가: A-

**강력한 바이럴 요소:**

1. **"XX귀신" 키워드 시스템** - 가장 강력한 바이럴 포인트. "나 돈귀신이래 ㅋㅋㅋ", "여자귀신 떴다;;" 같은 자연스러운 공유가 가능하다. 각자 다른 귀신 키워드가 나오므로 비교/공유 욕구를 자극한다. 결과 페이지에서 **XX귀신** 이 dark red (#8b0000) 볼드로 표시되어 시각적으로도 캡처에 적합하다.

2. **1부(공포)/2부(희망) 이중 구조** - "1부 무서웠는데 2부에서 갑자기 희망적이라 신기" 같은 반응이 나올 수 있는 구조다. 극적 반전이 대화 소재가 된다.

3. **경상도 사투리 할머니 캐릭터** - "그기 좀 무서운 기라..." 같은 독특한 문체가 밈(meme)화될 가능성이 높다. 사투리 자체가 한국 인터넷에서 밈으로 소비되는 문화적 코드와 맞닿아 있다.

4. **공포 + 사주의 장르 믹스** - 단순 사주도, 단순 공포도 아닌 조합이 화제성을 가진다. "사주인데 무섭다"는 컨셉 자체가 대화의 시작점이 된다.

**부족한 바이럴 요소:**

1. **공유 기능의 부재** - 결과 화면에 "공유하기" 버튼, 카카오톡 공유, 결과 이미지 저장 기능이 없다. 이것은 치명적인 누락이다. 바이럴 콘텐츠를 만들어놓고 공유 수단을 제공하지 않는 것은 불을 피워놓고 장작을 안 넣는 것과 같다.

2. **결과 요약 카드 부재** - "귀신사주 결과: [이름] - [XX귀신]" 같은 시각적 요약 카드(이미지)가 있으면 SNS 공유 시 클릭률이 크게 올라간다.

3. **비교 유도 장치 부재** - "친구의 사주도 봐주기" 같은 CTA가 없다. 로딩 리뷰에서 "친구들이랑 같이 봤는데"를 언급하면서도 실제로 친구에게 보내는 기능은 없다.

4. **OG 메타태그 최적화 미확인** - SNS에 링크를 공유했을 때 보이는 미리보기 이미지/제목/설명이 설정되어 있는지 확인 필요. 현재 metadata에 title과 description은 있으나 OpenGraph 이미지가 없다.

## 6. 재방문 유도 요소 분석

### 평가: C+

**현재 존재하는 재방문 요소:**

1. **"다시 보기" 버튼** - 결과 페이지 하단에 있어 다른 생년월일로 재시도 가능. 다만 자기 사주를 다시 보는 것은 동기가 약하고, "다른 사람 사주 보기"가 재방문의 주 동기다.

2. **리뷰 "자꾸 보게 됨"** - 로딩 리뷰에 재방문 암시가 있지만, 실제 재방문을 유도하는 메커니즘은 아니다.

**부재하는 재방문 요소:**

1. **주변인 사주 보기 유도** - "가족/친구/연인의 사주도 확인해보세요" CTA가 결과 페이지에 필요하다. 사주 서비스의 재방문 핵심 동력은 "내 사주" 이후 "주변인 사주"다.

2. **궁합 기능** - "상대방과의 궁합 보기" 같은 확장 기능이 있으면 재방문율이 크게 올라간다. 2명의 사주를 비교하는 것은 자연스러운 다음 단계다.

3. **시즌/이벤트 콘텐츠** - "2026년 신년 특별 운세", "올해의 귀신 경고" 같은 시즌 콘텐츠가 있으면 주기적 재방문이 발생한다.

4. **결과 저장/다시보기** - 현재는 결과를 저장하거나 나중에 다시 볼 수 있는 방법이 없다. 결과 URL 고유화 또는 로컬 저장 기능이 필요하다.

## 7. 캐릭터/세계관 깊이와 일관성

### 평가: B+

### 세계관 구조

| 요소 | 설정 | 일관성 |
|------|------|--------|
| 인물 | 고 김귀자 할머니 (1920~2007) | O - 인트로에서 명확히 설정 |
| 시대 | 1990년대 | O - 프롬프트에서 유지 |
| 장소 | 경상북도 청송 | O - 사투리 지시와 일치 |
| 능력 | 귀신사주 (공포+정확도) | O - 프롬프트에서 구현 |
| 톤 | 따뜻 + 으스스 | O - 프롬프트에서 명시 |

### 깊이 분석

**잘 구축된 부분:**
- "고(故)" 표현 + 생몰년으로 이미 돌아가신 분의 유산이라는 설정이 신비감과 경외감을 동시에 만든다.
- "생전 영상 보기" 버튼 + 실제 영상 재생은 세계관의 물리적 증거를 제공하여 몰입도를 극대화한다.
- AI 프롬프트에서 "할머니의 말투를 재현"이라는 명확한 프레이밍이 "AI가 사주를 보는 것"이 아닌 "할머니의 사주법을 AI가 전달하는 것"이라는 서사를 만든다.

**깊이가 부족한 부분:**

1. **할머니의 배경 서사 부재** - 김귀자 할머니가 어떻게 이 능력을 갖게 되었는지에 대한 이야기가 없다. "어릴 적 큰 병을 앓고 나서 신기가 열렸다", "꿈에서 조상이 나타나 가르침을 받았다" 같은 오리진 스토리가 있으면 세계관이 훨씬 풍부해진다. 인트로나 VideoScene 이전에 짧은 텍스트로 추가할 수 있다.

2. **장소(청송)의 활용 부족** - "경상북도 청송"이라는 설정은 인트로에서 한 번 언급되고 잊혀진다. 청송은 주왕산 국립공원이 있는 깊은 산골이다. "산 깊은 마을에서...", "안개가 자욱한 계곡 옆 작은 집에서..." 같은 장소 묘사가 프롬프트에 들어가면 할머니의 세계관이 더 입체적이 된다.

3. **일관성 깨지는 지점:**
   - 인트로의 톤(정중한 설명)과 프롬프트의 톤(반말 사투리)이 급격히 전환된다. 인트로도 세계관에 맞게 조정하면 전체 경험이 매끄러워진다.
   - 결과 화면이 밝은 배경(#fafaf5)으로 전환되면서 공포 분위기가 완전히 깨진다. 어두운 톤을 유지하면서 읽기 편한 디자인을 찾는 것이 세계관 유지에 중요하다.
